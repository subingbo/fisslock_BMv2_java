本项目在实验环境中用 BMv2（simple_switch） 代替真实 Tofino 交换机，运行 switch/p4_bmv2 的 P4 锁数据面。应用侧使用 Java（FissLockClient），通过 固定 gRPC 地址与端口（默认本机 50051）连接独立部署的 Java Sidecar（LockSidecarMain，通常需 sudo）；业务代码只调用 tryLock / unlock，不直接操作网卡。Sidecar 作为 锁协议的网络适配层：在 0.1 中通过 Scapy/pcap 向 veth-inject 发送 ACQUIRE、读取 GRANT；在 0.2 中可经 JNI 调用 C++ lib/lock.cc，与论文主机锁逻辑对齐。锁包格式与仓库内 C++ lib 一致，但 gRPC 两端均为 Java 进程，并非 C 语言 gRPC 客户端。
